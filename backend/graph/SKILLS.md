# Skills — graph (GraphQL)

## What This Directory Does
GraphQL API layer. Schema-first approach using `gqlgen` for code generation.

## Files
- **schema.graphqls** — Schema definition (types, queries, mutations). THE source of truth.
- **resolver.go** — `Resolver` struct holding all repository + hub dependencies.
- **schema.resolvers.go** — Hand-written resolver implementations. gqlgen generates stubs here.
- **generated.go** — Auto-generated by gqlgen. DO NOT EDIT.
- **model/models_gen.go** — Auto-generated Go types matching GraphQL types. DO NOT EDIT.

## Current Schema
```graphql
Query:
  campuses: [Campus!]!
  campus(id: ID!): Campus
  liveLocation(CampusId: ID!): LiveLocation

Mutation:
  changeCampus(studentId: ID!, toCampusId: ID!): Boolean!
```

## Resolver Dependencies
```go
type Resolver struct {
    CampusRepo  *repository.CampusRepo
    PilotRepo   *repository.PilotRepo
    StudentRepo *repository.StudentRepo
    Hub         *location.Hub
}
```

## How to Add a Query/Mutation
1. Edit `schema.graphqls` — add types + query/mutation fields
2. Run: `go run github.com/99designs/gqlgen generate`
3. gqlgen creates stub methods in `schema.resolvers.go` with `panic("not implemented")`
4. Replace the panic with your implementation
5. If you need new repos, add them to `Resolver` struct in `resolver.go` and wire in `cmd/server/main.go`

## Config
`gqlgen.yml` in backend root configures code generation paths. The `autobind` feature maps Go types to GraphQL types automatically.

## Auth
GraphQL endpoint (`POST /graphql`) is protected by `auth.Middleware(jwtSvc)` + `ProtectedLimiter` (200 req/min per user). Claims available via context.
The playground (`GET /playground`) has NO auth — it's a dev tool.
